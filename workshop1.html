Section 2 · 核心概念

什麼是 Agent？

16:15-16:45

Agent = 指令 + 護欄 + 工具

— OpenAI Definition

組成 說明 範例
指令 (Instructions) Agent 該做什麼 「你是 JIRA 專家，負責查詢和更新 issue」
護欄 (Guardrails) Agent 不該做什麼 「不能刪除 issue」「不能存取其他專案」
工具 (Tools) Agent 能做什麼 呼叫 JIRA API、查詢資料庫

Section 3 · MCP vs. A2A

MCP：解決 N×M 問題

16:45-17:15

Model Context Protocol

Anthropic 2024 年 11 月推出，現由 Linux Foundation 管理

解決什麼問題？

Agent 怎麼連接工具和資料？

沒有 MCP：

5 個 Agent × 10 種工具 = 50 個客製整合 ❌

有了 MCP：

5 + 10 = 15 個標準實作 ✓

讓各種設備能插上電腦

Section 3 · MCP vs. A2A

A2A：解決「自由開發」

16:45-17:15

Agent-to-Agent Protocol

Google 2025 年 4 月推出，150+ 組織支援，Linux Foundation 託管

解決什麼問題？

不同 Agent 之間怎麼溝通？
Agent 怎麼被平台發現和調度？

AAIF 採用 A2A 的意義：

  • 用 LangChain 開發？可以！
  • 用 Semantic Kernel？可以！
  • 用 CrewAI？可以！
  • 自己從頭寫？也可以！

類比：HTTP 標準
讓各種網站能互相連結

A2A 核心組件

Agent Card
/.well-known/agent.json

發布 Agent 能力的 JSON 檔案
讓 Supervisor 能「發現」你

📋
Tasks
Agent 間的工作單元
📦
Artifacts
交付物（文件、資料）

JSON-RPC 2.0 over HTTPS · 短期 Token 認證
